7.1 Historia de Windows

7.1.1 Sistema operativo de disco

Las computadoras no tenían discos duros ni memoria USB. Usaban tarjetas perforadas, cintas de papel y casetes para almacenar datos

Con la llegada de los disquetes y discos duros, se necesitó un software para gestionarlos. Así nació el Sistema Operativo de Disco (DOS), que organizaba archivos en los discos.

Microsoft compró DOS y creó MS-DOS, un sistema basado en comandos de texto para gestionar archivos y programas que permitía que las computadoras accedieran al disco y cargaran el sistema operativo.

7.1.2 Vulnerabilidades del sistema operativo

Los sistemas operativos pueden tener vulnerabilidades que los atacantes explotan para obtener acceso no autorizado, modificar permisos o manipular datos. Esto pone en riesgo la seguridad y el funcionamiento del sistema.

Protección y Seguridad en Windows

  • Protección contra malware: Windows Defender protege contra virus y amenazas. Si está desactivado, el sistema es más vulnerable.
  • Servicios desconocidos: Es clave identificar y administrar los servicios en segundo plano para evitar riesgos.
  • Cifrado: Proteger datos con encriptación evita que sean explotados, especialmente en dispositivos móviles.
  • Política de seguridad: Configurar y cumplir políticas de seguridad previene ataques.
  • Firewall: Windows Firewall controla la comunicación en la red; se debe revisar y actualizar periódicamente.
  • Permisos de archivos: Asignar permisos adecuados evita accesos no autorizados.
  • Contraseñas seguras: Usar contraseñas fuertes, sobre todo en cuentas de administrador.
  • Inicio de sesión como administrador: Se recomienda usar cuentas estándar y solo elevar privilegios cuando sea necesario.
7.1.3 GUI de Windows

Windows tiene una interfaz gráfica de usuario (Graphical User Interface, GUI) para que los usuarios trabajen con software y archivos de datos. La interfaz gráfica de usuario tiene un área principal que es conocida como El Escritorio, mostrada en la figura.

7.2 Arquitectura y operaciones de Windows

7.2.1 Capa de Abstracción de Hardware

Windows es compatible con distintos tipos de hardware, ya sea en computadoras compradas o ensambladas. Para funcionar, usa una Capa de Abstracción de Hardware (HAL), que gestiona la comunicación entre el hardware y el kernel.

El kernel es el núcleo del sistema operativo y controla la computadora, gestionando entrada/salida, memoria y periféricos. Aunque depende de la HAL, en algunos casos se comunica directamente con el hardware.

7.2.2 Modo de Usuario y Modo de Kernel

Modos de ejecución: Núcleo (kernel): Código del sistema operativo con acceso completo al hardware; errores pueden colapsar el sistema. Usuario: Ejecución de aplicaciones con acceso restringido; errores afectan solo la aplicación. Controladores y espacios de direcciones Pueden operar en ambos modos.

En núcleo se usa un único espacio de direcciones, mientras que en usuario cada aplicación tiene su espacio privado.

7.2.3 Sistema de Archivos de Windows

exFAT: Sistema simple, compatible con muchos SO. FAT presenta limitaciones (particiones, tamaños); FAT32 es el más usado.

HFS+: Sistema de Mac OS X, permite nombres y tamaños mayores. Windows puede leerlo con software especializado.

EXT: Usado en Linux. Windows lee particiones EXT con ayuda de software.

NTFS:

  • Principal en Windows; soportado también en Linux.
  • Mac OS X solo lee NTFS, pero puede escribir con drivers especiales.
  • Soporta archivos y particiones grandes, seguridad avanzada y encriptación.
  • Utiliza descriptores de seguridad y marcas de tiempo (MACE) para el control y seguimiento de archivos.

Formateo y particionado:

Antes de usar un disco, se particiona y formatea (usualmente con NTFS) para crear estructuras clave como el sector de arranque, la Master File Table (MFT), archivos del sistema y el área de archivos.

7.2.4 Flujo de Datos Alternativo (ADS)

NTFS almacena archivos como atributos (nombre, marca de hora, etc.), con los datos en el atributo DATA. Permite flujos de datos alternativos (ADS) para almacenar información extra, que puede ser usado maliciosamente para ocultar código. Los archivos con ADS se identifican con el nombre seguido de dos puntos, por ejemplo, Testfile.txt:ADS.

7.2.5 Proceso de Arranque de Windows

Firmware y arranque: Con BIOS, se hace POST, se localiza el MBR y se carga el sistema operativo. Con UEFI, se cargan archivos .efi desde la partición

ESP, mejorando la seguridad. Proceso de carga: Bootmgr.exe lee la base de datos de arranque (BCD).

Si la computadora estaba en hibernación, se usa Winresume.exe; si no, Winload.exe carga el registro y verifica los controladores.

Inicio del sistema: Ntoskrnl.exe inicia el núcleo de Windows y configura la HAL.

SMSS crea el entorno de usuario e inicia Winlogon para el acceso del usuario.

7.2.6 Procesos, subprocesos y servicios

Una aplicación se compone de procesos, cada uno con al menos un subproceso.

Los subprocesos comparten el mismo espacio de direcciones, aislados de otros procesos.

Windows permite multitarea, ejecutando múltiples subprocesos según la cantidad de procesadores.

Algunos procesos son servicios que operan en segundo plano y son esenciales para el sistema.

Los servicios se pueden configurar (iniciar, detener, reiniciar) mediante el panel de Servicios, y se debe tener cuidado con sus dependencias.

7.2.7 Asignación de Memoria y Controladores
La memoria RAM almacena instrucciones hasta que la CPU las procesa. Cada proceso tiene un espacio de direcciones virtuales: 4 GB en Windows 32 ‑bits y 8 TB en 64‑bits. El sistema traduce direcciones virtuales a físicas mediante tablas de página. Los procesos usan identificadores para acceder a recursos del núcleo sin conexión directa.RAMMap (de Sysinternals) permite visualizar cómo Windows asigna la memoria del sistema.
7.2.8 El Registro de Windows

Windows almacena información sobre hardware, aplicaciones, usuarios y configuración en una base de datos llamada el Registro. Este registro es jerárquico y organiza la información en subárboles, claves y subclaves. Los valores, que contienen los datos, se guardan en las claves y subclaves. Una clave puede tener hasta 512 niveles de profundidad. También se registra cómo interactúan estos objetos, como los archivos que abren las aplicaciones o los detalles de las propiedades de las carpetas.

HKEY_CURRENT_USER (HKCU) Contiene información sobre el usuario actualmente conectado.

HKEY_USERS (HKU) Contiene información sobre todas las cuentas de usuario en el host.

HKEY_CLASSES_ROOT (HKCR) Almacena información sobre la vinculación e incrustación de objetos (OLE), que permite integrar objetos de otras aplicaciones (por ejemplo, una hoja de cálculo en un documento de Word).

HKEY_LOCAL_MACHINE (HKLM) Contiene información relacionada con el sistema.

HKEY_CURRENT_CONFIG (HKCC) Contiene información sobre el perfil de hardware actual.

7.3 Configuración y monitoreo de Windows

7.3.1 Ejecutar como administrador

Evitar iniciar sesión con cuentas administrativas:

  • No se recomienda iniciar sesión con cuentas administrativas debido al riesgo de que el malware obtenga acceso completo a los archivos.
7.3.2 CLI y PowerShell

La CLI de Windows (cmd.exe) permite ejecutar programas, gestionar archivos y navegar por el sistema. Se pueden crear archivos por lotes para automatizar comandos. Se accede buscándolo en el sistema y puede ejecutarse como administrador para más control. PowerShell es una alternativa más avanzada que permite automatizar tareas mediante cmdlets, scripts (.ps1) y funciones. Se abre buscando "PowerShell" y también puede ejecutarse con privilegios de administrador.

Estos son los tipos de comandos que puede ejecutar PowerShell:

  • cmdlets - estos comandos realizan una acción y envían un resultado u objeto al siguiente comando que se ejecutará.
  • Scripts de PowerShell - estos son archivos con una extensión .ps1 que contienen comandos de PowerShell que se ejecutan.
  • Funciones de PowerShell - estas son fragmentos de código a los que se puede hacer referencia en un script.

7.3.3 Administrador de Tareas y Monitor de Recursos

El Administrador de tareas y otras herramientas de Windows ayudan a los administradores a monitorear aplicaciones, servicios y procesos. También permiten analizar el rendimiento del sistema (CPU, memoria, red) y detectar posibles problemas, como malware o fallos en componentes.

7.3.4 Accediendo a Recursos de Red

Windows usa redes para compartir archivos y recursos mediante SMB y UNC. Los recursos compartidos pueden tener permisos de acceso y algunos administrativos llevan $ en su nombre (ej. C$). Se puede acceder a ellos escribiendo la ruta UNC en el explorador de archivos.

Para controlar computadoras remotas, Windows usa RDP (Escritorio Remoto), pero debe manejarse con precaución ya que puede ser un objetivo de ataques. Se recomienda restringir su acceso y aplicar medidas de seguridad como Zero Trust.

7.4 Seguridad de Windows

7.4.1 El comando netstat

El comando netstat puede usarse para buscar conexiones entrantes o salientes no autorizadas. Cuando se cree que un programa puede ser malware, se puede investigar para determinar su legitimidad. Para facilitar el proceso, es posible vincular las conexiones con los procesos en ejecución en el Administrador de tareas.

7.4.2 windows defender

El malware incluye virus, gusanos, troyanos, registradores de teclado, spyware y adware. Estos están diseñados para invadir la privacidad, robar información y dañar la computadora o los datos.

7.4.3 Firewall windows defender

Un firewall deniega selectivamente el tráfico a una PC o a un segmento de red. Los firewalls suelen actuar abriendo y cerrando los puertos que utilizan diversas aplicaciones.

Se rechaza todo paquete que no esté explícitamente permitido.
